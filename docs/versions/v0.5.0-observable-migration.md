## 主题

SwiftUI 现代化 audit 的最大单项：把全仓 8 个 `ObservableObject + @Published + @StateObject/@ObservedObject` 老栈迁到 `@Observable + @Bindable`（macOS 14 已支持）。同时借迁移之机拆分 `UsageService.swift`（887 行 OAuth + token refresh + polling timer + backoff 一锅烩）。

涉及面：
- 8 个 service / model 类：`UsageService` / `UsageHistoryService` / `NotificationService` / `AppUpdater` / `UsageStatsService` / `ProviderCoordinator` / `ProviderRuntime` / `LaunchAtLoginModel`
- `MultiMenuBarLabel.swift:73-84` 的 `RuntimeAggregator` 手搓 Combine `sink → objectWillChange.send` 可自然消除
- 全部 view 端 `@StateObject` / `@ObservedObject` / `@EnvironmentObject` 改 `let / @Bindable`
- 单测 spy 注入路径要重做

由于改动面大、涉及 OAuth / token refresh / Sparkle 等敏感链路 conformance，立 spec 时**必须**：
1. 先 brainstorm 是否拆成多 spec 串行（按 service 边界 / 按 view 边界）
2. 走完整 G2 跨模型 design-review
3. 完整测试矩阵 + 24h beta 用户回访（v0.2.2 Sparkle beta 通道走起）
