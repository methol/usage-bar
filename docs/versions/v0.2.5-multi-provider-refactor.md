# v0.2.5 — Multi-Provider Refactor

## 主题

纯架构重构（用户对外可感知变化几乎为零，仅 Settings 多一个尚不可用的"主 provider"选择器占位）：把当前 Claude 写死的"活体用量"链路（`UsageService` + `UsageResponse`/`UsageBucket` + `MenuBarLabel` + `PopoverView` 用量区 + 趋势/pace）抽象成"一个 provider 一个 `UsageProvider` 实现 + 统一的 `ProviderUsageSnapshot` 形状 + 每 provider 一个 `ProviderRuntime` 状态容器 + 一个 `ProviderCoordinator` 协调器"。Claude 被改写成"恰好是当前唯一注册的 provider"，OAuth/refresh/多账号/backoff 等 Claude 特有外壳保留在 `ClaudeProvider` 内部。这是 [ADR 0005](../adr/0005-reopen-multi-provider-direction.md)"分两步走"的第 2 步打地基 —— 地基打好后，[v0.2.6 Codex provider](./v0.2.6-codex-provider.md) 才能干净地插进来而不重复一份平行代码。

不引入 descriptor / 宏 / 插件框架（ADR 0002/0005 已明确排除"照搬 CodexBar 30+ provider"那条路）。`UsageEventStore` / `UsageStatsService` / `JSONLCostParser`（v0.2.3 的成本热力图，已经是 per-provider）这次基本不动，只把它的 provider 枚举并入统一的 `ProviderID`。

## 含 spec

- `2026-05-12-multi-provider-refactor` — `UsageProvider` 协议 + `ProviderUsageSnapshot` 统一形状 + `ProviderRuntime`/`ProviderCoordinator` + Claude 改写成 provider + 视图层泛化 + `primaryProviderID` 设置
