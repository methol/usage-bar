  v0.2.4 — cli-pty-fallback：**永久不实施**（scope-cut）

  作者自治决策（2026-05-11，user 明确"非必须不要停下来"自治模式下）：不引入 PTY 调用 Claude CLI 子进程作为 OAuth 兜底。

  评估理由：
  - **v0.1.1 已覆盖核心 CLI 用户场景**：Claude CLI Keychain 凭证 bootstrap 让已装 Claude Code 的用户零配置登录；OAuth 主路径 + Keychain 兜底两条已稳定
  - **PTY 仅在"OAuth + Keychain 同时不可用"的极端 fallback 触发**：用户拒绝 Keychain ACL + 主 OAuth refresh 永久失败的概率极低
  - **CodexBar 实现需要 daemon watchdog 子进程**（CodexBarClaudeWatchdog）维持稳定 PTY；引入跨进程复杂度（IPC / 进程退出处理 / launchd 注册）违反"Swift 原生最小依赖"原则（ADR 0001）
  - **ANSI 解析脆弱**：依赖 `claude /usage` 输出格式；任何 CLI 版本升级改输出 layout 都破坏 parser；维护成本高
  - **替代方案充分**：用户极端场景可直接走原 PKCE sign-in 流程（已有 SetupView / CodeEntry）

  与 v0.2.3 cookie-fallback（user 明确决策）同款 scope-cut：通过保守路径优先，避免引入边际收益低 + 维护成本高的兜底逻辑。

  - 主路径：OAuth API（v0.1.0~）

  - 离线 cost：本地 JSONL（v0.1.2）
  - 多账号：accounts store（v0.1.3）
  - 失败兜底：用户重 PKCE sign-in（始终可用）
